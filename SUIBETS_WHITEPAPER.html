Core Betting Contract Functions

  • place_bet(): Lock user tokens in pool, create bet record
  • settle_bet(): Verify HMAC, execute payout or mark loss
  • cancel_bet(): Refund before settlement (admin only)
  • claim_winnings(): User confirms receipt of payout

6. Anti-Cheat System - HMAC-SHA256

Prevents oracle/admin from tampering with settlement results:

üõ°Ô∏è How It Works
  1. Admin generates HMAC hash: HMAC-SHA256(betId|eventId|outcome|result, ADMIN_SECRET)
  2. Hash submitted with settlement transaction to smart contract
  3. Smart contract recalculates expected hash
  4. If hashes don't match ‚Üí SETTLEMENT FAILS
  5. Prevents result manipulation completely

9. Placing Bets - Complete Flow

Step-by-Step Process

  1. User selects sporting event, market, and outcome
  2. Frontend validates user has sufficient balance
  3. Transaction created and sent to Sui wallet for signing
  4. Smart contract executes:
    • Validates bet parameters
    • Locks tokens in betting pool
    • Creates immutable bet record
    • Sets status to "pending"
  5. Walrus stores bet data permanently
  6. User sees confirmation with transaction hash
‚ö° Bet Lifecycle

Pending: Awaiting event settlement ‚Üí Won/Lost: Settlement executed ‚Üí Claimed: User collected winnings

10. Claiming Winnings

Automatic Settlement

  1. Event concludes in real-world (e.g., Liverpool 3-1 AFC)
  2. Oracle detects final result
  3. Backend processes all bets for that event
  4. For winning bets:
    • HMAC-SHA256 hash calculated
    • Smart contract called to settle
    • Payout transferred to user's wallet
    • Bet marked as "won"
  5. User sees "Won Bets" section in UI
  6. Clicks "Claim Winnings" to confirm receipt
